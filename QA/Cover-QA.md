# Cover QA

## Purpose
Reject cover images that violate Template001 or verified product facts.

## Usage
Apply this checklist to every Template001 output before Final QA.

- [ ] Template001 loaded and manifest identifies it.
- [ ] Product and primary verified value are understandable within one second.
- [ ] Logo uses the fixed token anchor and approved asset.
- [ ] Product coverage is within global limits.
- [ ] Title, subtitle and selling-point counts pass Text Rules.
- [ ] No unverified accessory, certification, compatibility or claim appears.

## Dependencies / Related Modules / Version
Template001 and Final-QA. Related to Cover prompt and ProductLock-QA. Version `2.0.0`.
