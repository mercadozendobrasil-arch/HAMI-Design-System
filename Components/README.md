# Components

Reusable image-layout components live here. Components consume only `Tokens/`, expose fixed inputs and slots, and must not embed product-specific content.

- Purpose: prevent layout duplication.
- Usage: compose templates from documented component contracts.
- Dependencies: `Tokens/`, `Brand/`, `Rules/`.
- Related modules: `Templates/`, `QA/`.
- Version: `2.0.0-dev`.
